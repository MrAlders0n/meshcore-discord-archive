# MeshCore Discord Archive

A searchable static archive of the MeshCore Discord, built from
[DiscordChatExporter](https://github.com/Tyrrrz/DiscordChatExporter) HTML output.

**Live site:** https://mralders0n.github.io/meshcore-discord-archive/
*(enable once GitHub Pages is turned on — see below)*

## What's here

- `MeshCore/` — raw HTML exports from DiscordChatExporter, one file per thread.
- `scripts/build.py` — parses those HTML files and emits JSON data for the site.
- `site/` — the static site (HTML + CSS + vanilla JS + MiniSearch).
  - `site/data/` — generated JSON (committed, but regenerated by CI on every push).
- `.github/workflows/pages.yml` — builds data and deploys `site/` to GitHub Pages.

## Using the site

- Left sidebar lists every archived thread, grouped by forum.
- Top search bar searches across every message.
- `user:name` limits results to a specific author. Combine with keywords:
  `user:ripplebiz repeater`. Quote phrases: `"time sync"`.
- The dropdown on the right picks an author from the full list.
- Click any message's author name to jump straight to their messages.
- Clicking a search result opens the thread at that message.

## Running locally

```bash
python3 scripts/build.py                         # regenerate site/data/
cd site && python3 -m http.server 8000           # serve (fetch() needs http)
```

Then open http://localhost:8000.

## Adding more exports

Drop more DiscordChatExporter `.html` files into a subfolder of `MeshCore/`
(the subfolder name becomes the forum/category), commit, and push. CI will
re-parse and redeploy.

## Deploying to GitHub Pages

1. Push to `main`.
2. In the repo settings → Pages, set **Source: GitHub Actions**.
3. The `pages.yml` workflow will build and deploy on every push.
