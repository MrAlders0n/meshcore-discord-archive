olanwe 2025-12-28 5:04 p.m.
Thanks for your feedback. Yes, I admit this is a bit confusing at the moment. The merged firmware contains the full image (including the bootloader) and is meant for initial flashing. For updates (i.e. if Meshcore is already running on the device), only the regular firmware is required (see readme.md). The “app only” switch can be used to extract the regular firmware from a merged image (in case you only have the merged version available but still want to perform an update). In normal use, this switch can simply be ignored. At the moment, the separation between patcher and firmware source is intentional, as it keeps things simple (at least on my side — I don’t have to implement an additional file selector and downloader). Since @recrof is already considering an implementation in the regular web flasher , I didn’t put too much effort into this functionality, as the patcher will likely become obsolete once that feature is available. I will add a version number and “dev” tag to the firmware files to make them easier to recognize among already downloaded files.
