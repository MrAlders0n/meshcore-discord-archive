NotLebedev 2026-04-08 6:35 a.m.
Some boards like lilygo t3s3 have slots for SD cards. This enables significantly larger than built in flash memory storage (practically limited by 4GB filesize limit of fat32). This opens possibility to have significantly larger limits for contacts, channels and offline queue as well as some new features like storing all recieved messages on sd card to make them available from any device. I started tinkering with meshcore firmware and identified areas that need to be worked on: 1. Hardware support. Some initialization and configuration code needs to be added to existing board firmwares. I have a working change for t3s3 that enables sd card as secondary fs https://github.com/NotLebedev/MeshCore/tree/t3s3-sd-card . It successfully writes contacts and channels to sd card, but otherwise logic is unchanged 2. Extending contacts, channels and offline queue to support filesystem storage. Currently each of this components resides entirely in memory in some pretty large buffers. Reworking them to support storing them on sd card or partially in memory/partially on sd card will actually allow to increase limits without being stopped by memory consumption. 3. Graceful support for sd cards being changed/removed. Lastly there needs to be some logic to support migration to/from sd card. Currently DataStore::migrateToSecondaryFS only supports migration to empty secondary fs. To support sd cards better some form of merging when sd card is already initialized needs to be added I intend to continue work on this proposal and would appreciate some guidance/feedback
