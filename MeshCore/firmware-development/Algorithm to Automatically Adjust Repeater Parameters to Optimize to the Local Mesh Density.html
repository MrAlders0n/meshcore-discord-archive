📚 Forums / firmware-development / Algorithm to Automatically Adjust Repeater Parameters to Optimize to the Local Mesh Density
Submarine_Man 2026-03-14 7:28 p.m.
Attached is a document that examines the various repeater parameters (rxdelay, txdelay, direct.txdelay, CR) and proposes a density metric to automatically adjust them to optimize a repeater for its current environment. By automating this, the entire mesh would improve (collisions would be reduced) and repeater owners that leave their repeaters at the non-optimum default values would have them automatically adjusted to an improved value. Comments for improvements and corrections would be greatly appreciated.

Aleksey M. 2026-03-22 12:21 p.m.
Nothing special, just SDRplay RSPdx R2 with an inexpensive Chinese SAW-filter. And I collect statistics on received messages using a USB companion and HomeAssistant with meshcore integration and automation that creates entities for the last prefixes from the packet path, as well as placing the corresponding SNR and RSSI in them. This data is accumulated with InfluxDB and visualized using Grafana. This looks quite complicated to someone unfamiliar with HomeAssistant and related solutions, but it allows you to get a useful result in a shorter time compared to developing a special application. Here is the link to the HomeAssistant integration site https://meshcore-dev.github.io/meshcore-ha/ And a link to a solution that saves the SNR of packets received from repeaters https://github.com/WJ4IoT/Meshcore-Home-Assistant-Solutions/blob/main/SNR_Repeater.md (edited)
