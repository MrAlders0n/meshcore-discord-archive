Just Stuff™ 2026-03-11 3:14 p.m.
Also to clarify a couple of points from your notes:
  • Double press is not intended to switch transports. On the single-button UI the intended mapping is:
single click = next page double click = previous page long press on BLE page = switch wireless transport to WiFi/TCP long press on WiFi page = toggle AP/client triple click on WiFi page = switch wireless transport back to BLE
  • USB is not a separate mode in this build. USB should remain available as the wired companion interface all the time, while set transport ble/tcp/wifi only changes the wireless side.
  • A normal serial terminal like CoolTerm will not talk to the runtime USB interface correctly, because the default USB interface is the framed companion protocol, not the raw text CLI. For normal USB testing you need meshcli or another client that speaks the companion protocol. The raw text CLI is only available through CLI Rescue during boot.
So if you retest after the next build, the main extra thing I’m interested in is whether V2 is still unexpectedly rebooting on double-click, since that is not supposed to happen.
