taedryn 2025-10-24 7:00 p.m.
i'm reaching the point where my BBS code is nearing ready to use, in an MVP sense -- it's to the point where i can run it for multiple days, and access it from a zero-hop node, but it's not quite ready for proper deployment yet. it's written in Python using meshcore_py, and currently works using a USB companion node as a "modem" where each DM session is a separate BBS connection. it's targeted for a Raspberry Pi Zero and an nRF based radio, so it can be solar powered. it's intentionally old-school, both for nostalgia reasons, and because it suits LoRa data sizes quite well. currently it supports an arbitrary number of rooms (discussion areas), private messages, and a simple privilege system. data is stored in a SQLite3 database. it's still missing many of the commands needed to be a truly functional system, while i work on making the MeshCore components work better in the real world. you can see the current state of development at https://github.com/taedryn/mesh-citadel. you can read more about Citadel BBSes via your favorite search engine, but they have a long history, the first system coming up in 1981 or so.

Pete 2026-01-10 12:37 p.m.
Forgive me if i'm telling you stuff you already know, but the way that meshcore radios work is that there is a "radio", and an "app". The radio and app communicate via something called "Companion Protocol." That protocol underneath is in fact "serial", but in order to interact with the radio the binary-based "Companion Protocol" is used to do that interaction. As part of the companion protocol, things like specified radio frequency, bandwidth, etc are available to be configured. So in this case "mesh-citadel" is an app that is utilizing a library that implements the Companion Protocol, and as such includes things like settable frequency as those are things that might need to be sent to a radio (for instance, if you plugged a fresh new meshcore device into it without configuring it first, citadel could theoretically handle all that setup for you based on the configuration values. Hope that helps!
